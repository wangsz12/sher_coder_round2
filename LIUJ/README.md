# Git、Markdown学习史  
### 一个落后小菜的~~快乐~~辛酸编程路  
* * *  
## 1.Git学习  
   ***过程***：我开始学习Git的时间从10月5日才刚刚开始，起步晚、时间紧，还要用我win7的台式退休机面对各种bug......总之一系列难题扑面而来让我有种西游记的既视感。不过所幸我坚持了下来并以俩天的速度完成了教程学习，弯道超车，不辱使命。  
   ***困难和收获***：从一开始就遭到“劝退”――Github在电脑上注册不了，忙活了一下午还是没有一点进展的我差点砸了那古董电脑，最后还是我灵机一动在手机上注册成功才化解危机。
                 到了学Git时难题更是多如牛毛，一开始下载费时自不必说，后面好多代码操作时几乎处处都有报错，有好多次都忍不住问学长学姐，但还是忍住了。因为我知道拿来的始终不是自己的，努力习得的才更不易忘却，事实也确实如此，当学习大约一半时。仿佛开了任督二脉，速度猛增不到半天学完；了教程：）  
  **习得**：学会了Git的基本使用规则和与Github的链接如：
 git config    /git  init    /git add    /git commit     等

## 2.Markdown学习
  ***过程***：在Git之后学习，由于有任督二脉加持。轻松习得，不过实践起来不是很熟练，毕竟只学了一会儿就上手了（苦笑）  
 ***困难和收获***：几个星号，用不用加号，留不留空格，留几个......这些问题比较绕，需要特别记忆；还要修改字体的方法都用类似编程的方法实现（和word有所不同）短时还无法适应。不过个人认为学成后应该较word简单。
